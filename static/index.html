💡 关于登录路径
完整登录地址 = {你的域名/IP}/login/…。 添加随机后缀后,机器人扫到 /login 直接 404, 找不到登录页就无从爆破密码。轮换会生成新的 12 字符后缀,旧地址立刻失效 (现有登录的 session 不受影响)。

启用 HTTPS — 4 步即可
使用 Caddy + Let's Encrypt 自动签发证书,永久免费
1
准备域名 + 设 A 记录
到你的 DNS 服务商 (阿里云 / Cloudflare / Namecheap / 等)， 添加一条 A 记录, 指向本机的公网 IP 加载中…
2
等 DNS 全球生效 (一般 1-10 分钟)
可以用 nslookup 你的域名 验证一下,看到返回的就是上面那个 IP 就 OK
3
在这里输入域名 + 点启用
后台会自动验证 DNS → 装 Caddy → 申 Let's Encrypt 证书 → 配反代 → 重启服务,全程 30 秒内
4
用新地址登录
完成后,你的后台地址就变成 https://你的域名/login, 浏览器地址栏会显示绿锁。用户名密码不变。
⚠️ 注意事项
  • 需要 VPS 的 80 端口和 443 端口没有被其他服务占用 (脚本会自动放行防火墙)
  • 首次申请 Let's Encrypt 证书需要 ~10 秒,刚切完域名打不开属正常,稍等一下重试
  • 切换 HTTPS 后,旧的 http://IP:8080/ 地址依然可访问 (仅作应急后门)
  • 如果失败需要撤销,SSH 进 VPS 跑 systemctl stop caddy,改 /etc/proxybox/config.yaml 里的 public_host 即可
  • 免费版 Let's Encrypt 证书 90 天到期,Caddy 会自动续期,你不用管

如何用

注册:点上方"注册当前设备",系统会弹 Touch ID,确认后这台 Mac 的指纹就能登录后台了。iPhone 通过 iCloud Keychain 同步,Face ID 也能登。

登录:访问当前 /admin/<TOKEN>/ 后台 URL,会跳到登录页;Passkey 可用于登录 session。

登出:点上方"登出"清除 session,下次打开后台会重新要 Touch ID。

应急:Passkey 私钥只在你 Mac / iPhone 的 Secure Enclave 里,服务器不存。两台设备都丢了 → 用 SSH 读取 /etc/proxybox/admin.password,然后用旧 URL 进后台。

⚠️ URL Token 应急后门
/admin/<TOKEN>/ 仍是后台地址的一部分;Passkey/密码负责建立 session

何时轮换:怀疑 URL 截图被拍 / 浏览器同步泄露 / 想彻底关闭 URL 后门。

轮换会:生成新 URL token → 改 /etc/proxybox/config.yaml,不需要重启服务;旧后台 URL 立即失效

不会影响:已注册的 Passkey, 已登录的 session, 任何订阅 URL (所有设备订阅照常)

注意:新 token 只显示一次,务必复制保存。后续继续使用新的 /admin/<TOKEN>/ 地址进入后台。
